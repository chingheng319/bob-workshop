📋 12 個內建工具概覽

Bob 提供 12 個核心工具，分為四大類別：

類別 工具 主要用途
檔案讀取 read_file 讀取檔案內容（支援行範圍）
list_files 列出目錄檔案
search_files 搜尋檔案內容（regex）
程式碼編輯 write_to_file 寫入完整檔案內容
apply_diff 精確修改程式碼片段
insert_content 在指定行插入內容
list_code_definition_names 列出程式碼定義（類別、函數等）
系統互動 execute_command 執行 CLI 命令
ask_followup_question 向使用者提問
attempt_completion 完成任務並回報結果
進階功能 switch_mode 切換 Bob 模式
update_todo_list 更新任務清單

📖 檔案讀取工具

1. read_file - 讀取檔案內容

用途：讀取檔案內容，支援行範圍讀取以節省 Token

基本用法：



  
    src/app.ts

進階用法（行範圍）：



  
    src/utils.ts
    1-15
    50-150

💡 最佳實踐

  • 大檔案務必使用行範圍，可節省 80-95% Token
  • 一次最多讀取 10 個檔案
  • 先用 list_code_definition_names 定位目標
  • 合併相近的行範圍（間隔 <10 行）

2. list_files - 列出目錄檔案

用途：瀏覽目錄結構，了解專案組織


src/
true

⚠️ 使用時機

  • ✅ 探索未知專案結構
  • ✅ 尋找特定類型檔案
  • ❌ 不要用來確認剛建立的檔案（等使用者確認）

3. search_files - 搜尋檔案內容

用途：使用正規表達式搜尋程式碼模式


src/
class.*Controller
*.java

💡 應用場景

  • 尋找特定 API 端點
  • 找出所有 TODO 註解
  • 搜尋安全漏洞模式
  • 定位特定函數呼叫

✏️ 程式碼編輯工具

4. apply_diff - 精確修改程式碼

用途：精確修改程式碼片段，節省 90%+ Token


src/app.py

<<<<<<< SEARCH
:start_line:15
-------
def calculate_total(items):
    total = 0
    for item in items:
        total += item
    return total
=======
def calculate_total(items):
    """Calculate total with 10% markup"""
    return sum(item * 1.1 for item in items)
>>>>>>> REPLACE

💡 關鍵要點

  • 必須完全匹配：SEARCH 區塊要與原始碼一模一樣
  • 包含行號：使用 :start_line:N 指定起始行
  • 批次修改：一次 diff 可包含多個 SEARCH/REPLACE 區塊
  • 優先選擇：大檔案修改首選此工具

5. write_to_file - 寫入完整檔案

用途：建立新檔案或完全重寫檔案


config.json

{
  "apiEndpoint": "https://api.example.com",
  "version": "1.0.0"
}

4

⚠️ 使用時機

  • ✅ 建立新檔案
  • ✅ 完全重寫小檔案
  • ❌ 修改大檔案（用 apply_diff）
  • ❌ 部分更新（用 apply_diff）

6. insert_content - 插入內容

用途：在指定行號插入新內容


src/utils.ts
1

import { sum } from './math';

💡 特殊用法

  • line: 1 - 插入到檔案開頭
  • line: 0 - 附加到檔案結尾
  • 適合新增 import、註解等單行內容

7. list_code_definition_names - 列出程式碼定義

用途：快速了解程式碼結構，不讀取完整內容


src/services/

💡 應用場景

  • 快速瀏覽專案架構
  • 定位目標函數位置
  • 了解類別結構
  • 配合 read_file 行範圍使用

💻 系統互動工具

8. execute_command - 執行命令

用途：執行 CLI 命令，如測試、建置、部署


npm test

切換目錄執行：


cd backend && python -m pytest

💡 常見用途

  • 執行測試：npm test, mvn test
  • 建置專案：npm run build, mvn package
  • 安裝依賴：npm install, pip install -r requirements.txt
  • Git 操作：git status, git diff

9. ask_followup_question - 向使用者提問

用途：需要更多資訊時向使用者提問


API 端點的基礎 URL 是什麼？

  https://api.example.com
  http://localhost:8080

⚠️ 使用原則

  • 只在真正需要時提問
  • 提供具體的選項建議
  • 避免問可以從程式碼推斷的問題

10. attempt_completion - 完成任務

用途：任務完成時回報結果



✅ API 端點已建立
✅ 測試通過
✅ 文件已更新

💡 最佳實踐

  • 簡潔列出完成項目
  • 確認所有工具操作成功後才使用
  • 不重複任務描述，聚焦結果

🚀 進階功能工具

11. switch_mode - 切換模式

用途：在不同 Bob 模式間切換


code
需要進行程式碼實作

💡 模式選擇

  • plan - 規劃與設計階段
  • code - 程式碼實作階段
  • advanced - 使用 MCP 功能

12. update_todo_list - 更新任務清單

用途：管理多步驟任務的進度



[x] 分析需求
[-] 實作核心邏輯
[ ] 撰寫測試
[ ] 更新文件

💡 狀態標記

  • [ ] - 待處理
  • [-] - 進行中
  • [x] - 已完成

🎯 工具組合策略

常見工作流程

場景 1：新增功能

  1. list_code_definition_names - 了解現有結構
  2. read_file (行範圍) - 讀取相關程式碼
  3. apply_diff - 修改現有檔案
  4. write_to_file - 建立新檔案
  5. execute_command - 執行測試
  6. attempt_completion - 完成任務

場景 2：Bug 修復

  1. search_files - 搜尋錯誤模式
  2. read_file (行範圍) - 讀取問題區域
  3. apply_diff - 修正程式碼
  4. execute_command - 驗證修復
  5. attempt_completion - 回報結果

場景 3：重構

  1. list_code_definition_names - 分析結構
  2. read_file (多檔案) - 批次讀取
  3. apply_diff (多區塊) - 批次修改
  4. execute_command - 執行測試
  5. attempt_completion - 完成重構

💪 實作練習

練習 1：檔案操作組合

任務：在專案中新增一個工具函數

  1. 使用 list_code_definition_names 找出 utils 檔案
  2. 使用 read_file 讀取檔案結構
  3. 使用 apply_diff 新增函數
  4. 使用 execute_command 執行測試

練習 2：批次修改

任務：更新多個檔案的 API 端點

  1. 使用 search_files 找出所有使用舊端點的檔案
  2. 使用 read_file 批次讀取（最多 10 個）
  3. 使用 apply_diff 批次修改
  4. 使用 execute_command 驗證變更

練習 3：Token 優化

任務：修改大型檔案中的單一函數

  1. 使用 list_code_definition_names 定位函數
  2. 使用 read_file 只讀取函數所在行範圍
  3. 使用 apply_diff 精確修改
  4. 比較與 write_to_file 的 Token 差異

✨ 最佳實踐總結

🎯 工具選擇原則

  • 讀取優先：先用 list_code_definition_names 定位，再用 read_file 行範圍
  • 編輯優先：大檔案用 apply_diff，小檔案或新檔案用 write_to_file
  • 批次操作：盡可能在一次工具呼叫中完成多個操作
  • Token 意識：時刻考慮 Token 消耗，選擇最經濟的方案

⚠️ 常見錯誤

  • ❌ 讀取整個大檔案而不使用行範圍
  • ❌ 用 write_to_file 修改大檔案
  • ❌ 重複讀取已在上下文中的檔案
  • ❌ apply_diff 的 SEARCH 區塊不完全匹配
  • ❌ 忘記在 apply_diff 中指定 :start_line:
