安全掃描
內建安全掃描 — 機密、上下文分析、CVE 掃描
💬 談話要點
  • 現在我們知道應用程式運作正常 — 測試通過 — 我們需要知道它也可以安全發布。Bob 具有內建的安全掃描功能，直接嵌入到開發人員工作流程中 — 不是在發布週期結束時的事後想法，而是在 IDE 中編寫程式碼時就在這裡。
  • Bob 具有內建的安全掃描功能，並在 Bob Findings 中呈現所有安全見解，按檔案組織，具有行級詳細資訊和可操作的修復建議。
💬 談話要點
您在那裡看到的不僅僅是錯誤清單 — 這是 Bob 的上下文分析：推理程式碼語義、標記反模式，並建議具體的重構步驟。例如，Bob 標記了 PaymentService.java 並建議在 Transaction 實體中僅儲存遮罩的卡號，而不是完整的卡號。
💡 提示
從自動化測試到安全掃描，Bob 解決了正確性和安全性 — 這些領域通常需要單獨的工具、團隊和 SDLC 中的後續檢查點。

這就是左移在實踐中的樣子：問題在開發時間早期就被發現和解決，甚至在任何程式碼到達 CI/CD 流水線之前。

請記住：Bob 的安全掃描是額外的層級，而不是現有 SAST、DAST 或漏洞管理工具的替代品。您的企業安全流程仍然適用 — Bob 在程式碼離開開發人員的機器之前更早地捕獲問題。

Scan the payment-app-java17 application for security issues in two phases: Phase 1 — Quick scan: - Hardcoded secrets, API keys, passwords, tokens in source and config Phase 2 — Contextual analysis: - Logic flaws and insecure patterns - Missing input validation - Anti-patterns that introduce security risk For each issue found: show file, line number, severity, and a one-line fix recommendation. Produce a report: SecurityScanReport.md
💬 談話要點
使這變得強大的不僅僅是 Bob 發現了問題 — 報告包含每個發現的建議修復。開發人員不必弄清楚如何修復它，Bob 會準確地告訴他們該做什麼。如果他們想進一步，他們可以簡單地要求 Bob 修復報告中列出的問題 — 就在 IDE 中，無需切換上下文。
