💾 7.1 什麼是 Memory Import？

定義

Memory Import（記憶匯入）是將重要資訊預先載入到 Bob 的上下文（Context）中，讓 Bob 在對話過程中能夠記住並使用這些資訊。

為什麼需要 Memory Import？

  • 避免重複說明：不需要每次都重新解釋專案背景
  • 提升效率：Bob 能快速理解上下文
  • 節省 Token：比每次讀取檔案更經濟
  • 保持一致性：確保 Bob 使用正確的資訊

💡 Memory Import vs 讀取檔案

讀取檔案（read_file）

  • 每次對話都要重新讀取
  • 消耗較多 Token
  • 適合：需要最新內容的檔案

Memory Import

  • 一次匯入，持續使用
  • 節省 Token
  • 適合：不常變動的重要資訊

⏰ 7.2 何時使用 Memory Import？

✅ 適合匯入的內容

  • 專案規範：編碼標準、命名慣例
  • API 文件：常用的 API 說明
  • 架構設計：系統架構圖、模組說明
  • 業務規則：核心業務邏輯
  • 技術決策：為什麼選擇某個技術
  • 常見問題：FAQ 與解決方案

❌ 不適合匯入的內容

  • 程式碼檔案：會頻繁變動
  • 大量資料：會消耗太多 Token
  • 臨時資訊：只用一次的內容
  • 敏感資料：密碼、金鑰等
  • 過時資訊：已經不適用的內容

💡 判斷原則

如果某個資訊符合以下條件，就適合使用 Memory Import：

  1. 會在多次對話中重複使用
  2. 內容相對穩定，不常變動
  3. 對理解專案很重要
  4. 大小適中（不要太大）

🛠️ 7.3 如何使用 Memory Import？

在 Bob Shell 中使用

Bob Shell 提供 /memory 指令來管理記憶體：

# 查看當前記憶體狀態
/memory

# 匯入檔案到記憶體
/memory import path/to/file.md

# 清除記憶體
/memory clear

在 Bob IDE 中使用

在 Bob IDE 中，可以透過對話方式請求匯入：

請將 .bob/rules/coding-standards.md 的內容匯入記憶體，
我接下來的開發都需要遵循這些規範。

💡 自動匯入

某些檔案會自動被 Bob 載入：

  • .bob/AGENTS.md - 專案指引
  • .bob/rules/*.md - 專案規則
  • .bob/skills/*.md - 自訂技能

這些檔案不需要手動匯入，Bob 會自動讀取。

📊 7.4 記憶體管理策略

Context 大小限制

Bob 的 Context 有大小限制（Token 限制），需要謹慎管理：

  • 對話歷史：佔用 Context
  • 已讀取的檔案：佔用 Context
  • Memory Import：佔用 Context
  • AGENTS.md 等：自動載入，佔用 Context
策略 說明 適用場景
精簡內容 只匯入最重要的資訊 Context 接近上限時
分批匯入 依任務需求匯入不同內容 有多個不同類型的任務
定期清理 完成任務後清除記憶體 開始新任務時
優先順序 先匯入最重要的資訊 資訊量大時

⚠️ Context 管理原則

  1. 監控使用量：注意 Context 使用情況
  2. 適時清理：完成任務後清除不需要的內容
  3. 精簡文件：移除冗餘資訊
  4. 分段處理：大任務分成小任務，分別管理 Context

🎯 7.5 實戰範例

範例 1：匯入編碼規範

情境：開始開發新功能，需要遵循專案規範

# 在 Bob Shell 中
/memory import .bob/rules/coding-standards.md

# 在 Bob IDE 中
請將編碼規範（.bob/rules/coding-standards.md）匯入記憶體，
接下來我要開發使用者管理功能，請確保程式碼符合規範。

範例 2：匯入 API 文件

情境：需要整合外部 API，頻繁參考文件

請將支付 API 文件（docs/payment-api.md）匯入記憶體，
我需要實作支付功能，會頻繁使用這些 API。

範例 3：清除記憶體

情境：完成支付功能，要開始開發訂單功能

# 在 Bob Shell 中
/memory clear

# 在 Bob IDE 中
請清除之前匯入的支付 API 文件，
現在要開始開發訂單功能了。

✏️ 7.6 實戰練習

練習 1：匯入專案規範

練習使用 Memory Import：

  1. 準備一個規範文件（如 coding-standards.md）
  2. 使用 Memory Import 匯入
  3. 請 Bob 產生一段程式碼
  4. 檢查是否符合匯入的規範

練習 2：管理 Context

練習 Context 管理：

  1. 匯入多個文件到記憶體
  2. 觀察 Context 使用情況
  3. 完成任務後清除記憶體
  4. 開始新任務，匯入新的內容

練習 3：優化策略

實驗不同的匯入策略：

  1. 比較「每次讀檔」vs「Memory Import」的效率
  2. 測試匯入不同大小的文件
  3. 找出最適合你專案的策略

💡 7.7 Memory Import 最佳實踐

使用建議

  • 選擇性匯入：只匯入真正需要的內容
  • 保持精簡：文件內容要簡潔明確
  • 定期更新：確保匯入的是最新資訊
  • 適時清理：完成任務後清除記憶體
  • 監控使用：注意 Context 使用情況

文件準備

  • 結構化：使用清楚的標題與章節
  • 重點突出：重要資訊放在前面
  • 範例豐富：提供具體的程式碼範例
  • 避免冗餘：移除不必要的內容

⚠️ 常見錯誤

  • ❌ 匯入過多內容，超過 Context 限制
  • ❌ 匯入會頻繁變動的檔案
  • ❌ 忘記清除不需要的記憶體
  • ❌ 匯入的文件內容過時
  • ❌ 沒有監控 Context 使用情況
