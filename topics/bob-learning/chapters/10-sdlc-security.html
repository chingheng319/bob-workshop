🚫 .bobignore 檔案管理

什麼是 .bobignore？

.bobignore 是一個配置檔案，用於指定 Bob 不應該讀取或處理的檔案和目錄。 這對於保護敏感資料和優化效能至關重要。

⚠️ 為什麼需要 .bobignore？

  • 安全性：防止 Bob 讀取敏感資料（密碼、金鑰、個資）
  • 效能：避免載入不必要的大型檔案
  • 成本：減少 Token 消耗
  • 隱私：保護商業機密和專有資訊

💡 配置原則

  • 預設排除所有敏感檔案
  • 排除第三方依賴
  • 排除建置產物
  • 排除大型資料檔案
  • 定期審查和更新

建立 .bobignore 的步驟

  1. 在專案根目錄建立 .bobignore 檔案 
    touch .bobignore
  2. 參考 .gitignore 內容

    .bobignore 通常應該包含 .gitignore 的所有內容，再加上額外的敏感檔案

  3. 新增專案特定的排除規則

    根據專案類型新增特定的排除模式

  4. 測試驗證

    確認 Bob 不會讀取被排除的檔案

⚠️ 安全紅線

絕對不要讓 Bob 處理以下內容：

  • ❌ 生產環境的密碼或金鑰
  • ❌ 真實的個人資料或信用卡資訊
  • ❌ 未加密的敏感配置檔案
  • ❌ 客戶的專有程式碼（未經授權）
  • ❌ 安全漏洞的詳細資訊（在修復前）

✅ 安全最佳實踐

  • 環境分離：開發環境使用假資料
  • 最小權限：只給 Bob 必要的檔案存取權
  • 定期審查：檢查 Bob 的存取記錄
  • 加密儲存：敏感資料必須加密
  • 版本控制：敏感檔案不進入 Git

👀 程式碼審查最佳實踐

使用 /review 指令

Bob 提供 /review 指令進行程式碼審查：

/review src/services/UserService.java

💡 審查重點

  • 安全性：SQL Injection、XSS、CSRF 等漏洞
  • 效能：N+1 查詢、記憶體洩漏、演算法複雜度
  • 可維護性：程式碼結構、命名規範、註解品質
  • 最佳實踐：設計模式、SOLID 原則、DRY 原則

自訂安全掃描 Slash Command

建立專門的安全掃描指令：

檔案位置：.bob/slash-commands/security-scan.md

# Security Scan

執行全面的程式碼安全掃描，檢查以下項目：

## 檢查項目

1. **輸入驗證**
   - 檢查所有使用者輸入是否經過驗證
   - 確認資料型別和範圍檢查

2. **SQL Injection**
   - 檢查是否使用參數化查詢
   - 避免字串拼接 SQL

3. **XSS 防護**
   - 檢查輸出是否經過編碼
   - 確認 HTML/JavaScript 注入防護

4. **認證授權**
   - 檢查認證機制
   - 確認權限控制

5. **敏感資料**
   - 檢查是否有硬編碼的密碼或金鑰
   - 確認敏感資料是否加密

6. **錯誤處理**
   - 檢查是否洩漏敏感資訊
   - 確認錯誤訊息適當

## 輸出格式

- 列出發現的安全問題
- 標註嚴重程度（高/中/低）
- 提供修復建議

💡 使用方式

/security-scan src/controllers/

👥 團隊協作規範

建立團隊 Bob 使用規範

檔案位置：.bob/TEAM_GUIDELINES.md

# Bob 使用規範

## 安全規範

1. **禁止事項**
   - ❌ 不得讓 Bob 處理生產環境密碼
   - ❌ 不得上傳真實個人資料
   - ❌ 不得分享包含敏感資訊的對話

2. **必須事項**
   - ✅ 使用前檢查 .bobignore
   - ✅ 定期清理 Bob 上下文
   - ✅ 審查 Bob 產生的程式碼

## 開發流程

1. **需求階段**：使用 Plan 模式分析
2. **設計階段**：產生架構文件
3. **開發階段**：使用 Code 模式實作
4. **審查階段**：使用 /review 檢查
5. **部署階段**：產生部署文件

## 程式碼品質

- 遵循專案 Rules（.bob/rules/）
- 使用 /security-scan 檢查安全性
- 確保測試覆蓋率 >80%
- 產生的程式碼必須經過人工審查

## Token 優化

- 優先使用行範圍讀取
- 批次操作減少互動
- 保持回應簡潔
- 避免重複讀取檔案

💡 團隊協作建議

  • 統一規範：全團隊使用相同的 Rules 和 AGENTS.md
  • 知識分享：分享有效的 Slash Commands 和 Skills
  • 定期審查：檢討 Bob 使用效果和問題
  • 持續改進：根據經驗更新使用規範
