測試
5 分鐘
產生測試
產生涵蓋正常路徑、邊緣案例和失敗情境的完整測試套件
💬 談論重點
  • 測試是任何軟體交付生命週期中的核心環節 —— 像是持續整合／持續交付（CI/CD）、DevSecOps，以及敏捷開發等業界標準，都要求在程式碼發佈前先完成驗證。我們會讓 Bob 自動生成一套完整的測試案例，涵蓋正常情境、邊界條件與錯誤情境，完全不需要手動撰寫測試。
Prompts
2a · 產生測試
▼
Generate JUnit 5 integration tests for the payment API. Cover: successful authorization, declined card, capture after auth, refund after capture, invalid card number, duplicate refund attempt. Use MockMvc. Assert HTTP status codes and response body fields. Target 80% code coverage. Use Java 17. Generate tests for the payment-app-java17 directory. Generate a report with name PaymentApiIntegrationTestCoverageReport.md.
💡 提示
JaCoCo（Java Code Coverage）— 一個測量測試執行了多少程式碼的函式庫。它產生涵蓋率百分比報告。MockMvc — 一個 Spring 測試框架，讓您可以模擬對 REST 端點的 HTTP 請求，而無需啟動真實伺服器，使測試快速且隔離。
