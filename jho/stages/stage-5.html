Day-2 SRE 維運
RefreshPOD、滾動重啟、快取、憑證、Vault 導入
💬 談話要點 · #31
Day-2 維運通常依賴部落知識 — 知道正確的 oc 命令、旗標、順序和安全檢查。在凌晨 2 點，這會成為風險。

有了 Bob，維運變成意圖驅動。您不需要記住命令，只需描述您需要完成什麼。

Bob 不只是執行命令 — 它會觀察結果、驗證它，並在出錯時告訴您。

它消除了認知負擔，讓每個人都能安全地進行生產維運。
📝 簡報者備註 · #32
這是直接從終端機示範 BobShell 的絕佳時機。

從終端機會話：

1. 登入您的 OpenShift 叢集。

2. 輸入 bob 啟動 BobShell。

3. 貼上下面的提示，並觀察 Bob 如何即時執行 Day-2 維運任務。

這是一種簡單的方式，展示如何以對話方式處理例行的 SRE 活動，例如 pod 刷新、滾動重啟、快取失效、憑證輪換或 Vault 導入，而無需切換工具或手動拼接命令。
💡 提示 · #54
您不必全部執行 — 只需執行一兩個就足以示範此功能。
提示
5a · RefreshPOD
▼
Refresh the pods for payment-app in bob-demo-staging namespace. Delete pods one at a time with a 30-second grace period. Wait for each replacement pod to pass readiness checks before deleting the next one. Report status after each pod. Use oc commands.
5b · 滾動重啟
▼
Perform a rolling restart of payment-app in bob-demo-staging namespace. Use oc rollout restart. Monitor the rollout status. Confirm zero downtime by checking that the PodDisruptionBudget is respected and at least 2 pods are available at all times.
5c · 快取刷新
▼
Clear the local cache on all payment-app pods in bob-demo-staging namespace. For each pod, call POST /admin/cache/clear endpoint (using the most appropriate method: exec, port-forward, or direct service access). Report the HTTP status code and response body from each pod in a table format. If any pod returns a non-200 response, flag it and suggest specific remediation steps.
5d · 憑證更新 · [驗證進行中]
▼
The TLS certificate for payment-service-tls in bob-demo namespace is expiring in 7 days. Generate a renewal script that: 1. Creates a new self-signed cert (RSA 2048, 365 day validity) 2. Validates the new cert with openssl 3. Updates the OpenShift TLS secret 4. Triggers a rolling restart to pick up the new cert 5. Verifies the new cert is served by checking the Route
5e · Vault 導入 · [驗證進行中]
▼
A new client "acme-corp" needs to be onboarded to our payment platform. Using HashiCorp Vault, generate the onboarding script that: 1. Creates a Vault path: secret/clients/acme-corp/ 2. Generates and stores: API key, DB password, encryption key 3. Creates a Vault policy scoping read access to only acme-corp secrets 4. Binds the policy to the payment-service service account in OpenShift 5. Restarts payment-service to pick up the new Vault path
