🎯 5.1 什麼是 Skills？

定義

Skills（技能）是 Bob 可以執行的特定能力或功能模組。透過 Skills，你可以擴展 Bob 的功能，讓它能夠執行更多專業化的任務。

Skills 的特色

  • 模組化：每個 Skill 專注於特定功能
  • 可重用：Skill 可在不同專案中使用
  • 可組合：多個 Skills 可以組合使用
  • 可擴展：可以建立自訂 Skills

💡 Skills vs Tools

Tools（工具）：Bob 的基本操作能力

  • 例如：read_file, write_to_file, execute_command
  • 內建且固定，無法修改

Skills（技能）：更高層次的功能模組

  • 例如：程式碼審查、測試生成、文件產生
  • 可以自訂與擴展
  • 通常會組合使用多個 Tools

📊 5.2 AgentSkills 支援表

什麼是 AgentSkills？

AgentSkills 是 Bob 內建支援的技能集合。不同的 Agent（AI 模型）支援不同的 Skills。

Skill 類別 說明 常見用途
程式碼分析 分析程式碼結構與品質 程式碼審查、重構建議
測試生成 自動產生測試程式碼 單元測試、整合測試
文件產生 產生技術文件 API 文件、註解、README
程式碼轉換 轉換程式碼格式或語言 重構、遷移、現代化
安全掃描 檢查安全漏洞 漏洞檢測、安全審查

💡 如何查看支援的 Skills

在 Bob Cheatsheet 中可以找到完整的 AgentSkills 支援表，列出每個 Agent 支援哪些 Skills。

🛠️ 5.3 建立自訂 Skill

Skill 檔案結構

自訂 Skills 放在 .bob/skills/ 目錄：

專案根目錄/
├── .bob/
│   └── skills/
│       ├── code-review.md
│       ├── api-doc-generator.md
│       └── performance-analyzer.md
└── src/

Skill 檔案格式

  • 格式：Markdown (.md)
  • 命名：描述性名稱，小寫加連字符
  • 內容：描述 Skill 的功能、步驟、範例

📝 Skill 文件範本

# Skill 名稱

## 目的
說明這個 Skill 的用途

## 功能描述
詳細描述 Skill 的功能

## 執行步驟
1. 步驟一
2. 步驟二
3. 步驟三

## 輸入
說明需要什麼輸入

## 輸出
說明會產生什麼輸出

## 使用範例
提供具體的使用案例

## 注意事項
特殊情況或限制

🎯 5.4 實戰範例：程式碼審查 Skill

範例：code-review.md

# 程式碼審查 Skill

## 目的
對程式碼進行全面的品質審查，提供改善建議

## 功能描述
這個 Skill 會分析程式碼並檢查以下項目：
- 程式碼風格與規範
- 潛在的 Bug 或錯誤
- 效能問題
- 安全漏洞
- 可讀性與可維護性
- 測試覆蓋率

## 執行步驟
1. 讀取指定的程式碼檔案
2. 分析程式碼結構
3. 檢查是否符合專案規範（參考 .bob/rules/）
4. 識別潛在問題
5. 提供具體的改善建議
6. 產生審查報告

## 輸入
- 檔案路徑或目錄路徑
- 可選：審查重點（如 "安全性"、"效能"）

## 輸出
審查報告，包含：
- 問題清單（依嚴重程度分類）
- 每個問題的詳細說明
- 改善建議與範例程式碼
- 整體評分

## 使用範例

### 範例 1：審查單一檔案
```
請使用程式碼審查 Skill 檢查 src/main/java/com/example/UserService.java
```

### 範例 2：審查整個目錄
```
請使用程式碼審查 Skill 檢查 src/main/java/com/example/ 目錄，
重點關注安全性問題
```

## 審查標準
參考以下規則檔案：
- .bob/rules/coding-standards.md
- .bob/rules/security-rules.md
- .bob/rules/performance-guidelines.md

## 報告格式

### 嚴重問題 🔴
- 問題描述
- 位置：檔案名稱:行號
- 建議修改

### 警告 🟡
- 問題描述
- 位置：檔案名稱:行號
- 建議修改

### 建議 🔵
- 改善建議
- 位置：檔案名稱:行號
- 範例程式碼

## 注意事項
- 審查結果基於專案的 Rules 設定
- 如果沒有 Rules，使用通用最佳實踐
- 大型專案建議分批審查

📚 5.5 實戰範例：API 文件生成 Skill

範例：api-doc-generator.md

# API 文件生成 Skill

## 目的
自動產生 RESTful API 的完整文件

## 功能描述
分析 Controller 類別，產生包含以下內容的 API 文件：
- API 端點列表
- 請求方法（GET/POST/PUT/DELETE）
- 請求參數
- 請求體格式
- 回應格式
- 狀態碼說明
- 使用範例

## 執行步驟
1. 讀取 Controller 類別
2. 分析 @RequestMapping 等註解
3. 識別所有 API 端點
4. 提取參數與回應類型
5. 產生 Markdown 格式文件
6. 包含 curl 使用範例

## 輸入
- Controller 類別路徑
- 可選：輸出檔案路徑

## 輸出
Markdown 格式的 API 文件

## 使用範例

```
請使用 API 文件生成 Skill 為 
src/main/java/com/example/controller/UserController.java 
產生文件，輸出到 docs/api/user-api.md
```

## 文件範本

```markdown
# User API 文件

## 基本資訊
- Base URL: `/api/users`
- 版本: v1

## API 端點

### 1. 取得所有使用者
- **端點**: `GET /api/users`
- **說明**: 取得所有使用者列表
- **參數**: 
  - `page` (optional): 頁碼，預設 0
  - `size` (optional): 每頁筆數，預設 20
- **回應**: 
  ```json
  {
    "content": [...],
    "totalElements": 100,
    "totalPages": 5
  }
  ```
- **使用範例**:
  ```bash
  curl -X GET "http://localhost:8080/api/users?page=0&size=20"
  ```

### 2. 取得單一使用者
- **端點**: `GET /api/users/{id}`
- **說明**: 根據 ID 取得使用者
- **路徑參數**:
  - `id`: 使用者 ID
- **回應**:
  ```json
  {
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com"
  }
  ```
- **使用範例**:
  ```bash
  curl -X GET "http://localhost:8080/api/users/1"
  ```
```

## 注意事項
- 確保 Controller 有適當的註解
- 支援 Spring Boot 和 JAX-RS 註解
- 產生的文件可直接用於專案文件

✏️ 5.6 實戰練習

練習 1：建立第一個 Skill

建立一個簡單的 Skill：

  1. 建立 .bob/skills/ 目錄
  2. 建立 test-generator.md
  3. 定義測試生成的步驟與規則
  4. 測試 Skill 是否正常運作

練習 2：使用 Skill

實際使用你建立的 Skill：

  1. 選擇一個現有的類別
  2. 請 Bob 使用你的 Skill 產生測試
  3. 檢查產生的測試是否符合預期
  4. 根據結果調整 Skill 定義

練習 3：組合多個 Skills

建立一個工作流程，組合使用多個 Skills：

  1. 先使用程式碼審查 Skill
  2. 根據審查結果進行重構
  3. 使用測試生成 Skill 建立測試
  4. 使用文件生成 Skill 更新文件

💡 5.7 Skills 最佳實踐

Skill 設計原則

  • 單一職責：每個 Skill 專注於一個功能
  • 清楚的輸入輸出：明確定義需要什麼、產生什麼
  • 步驟明確：詳細描述執行步驟
  • 提供範例：包含實際使用案例
  • 可組合：設計時考慮與其他 Skills 的配合

Skill 命名建議

  • 使用動詞-名詞格式：generate-tests
  • 描述性且具體：api-doc-generator
  • 避免太通用：不要用 helper 或 utility
  • 保持一致性：團隊使用統一的命名風格

⚠️ 常見錯誤

  • ❌ Skill 功能太複雜，難以理解
  • ❌ 沒有明確的執行步驟
  • ❌ 缺少使用範例
  • ❌ 與 Rules 或 AGENTS.md 內容重複
  • ❌ Skill 之間有依賴但沒有說明
