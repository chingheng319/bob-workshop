🎨 2.1 Mode 系統

什麼是 Mode？

Bob 提供多種工作模式（Mode），每種模式針對不同的開發任務優化。選擇正確的模式能大幅提升效率。

📝 Plan 模式

用途：規劃、設計、分析

  • 適合場景：
    • 需求分析與架構設計
    • 技術方案評估
    • 問題診斷與分析
    • 產生設計文件
  • 特色：
    • 專注於思考與規劃
    • 不直接修改程式碼
    • 提供詳細的分析報告

💻 Code 模式

用途：實作、修改、除錯

  • 適合場景：
    • 撰寫新功能
    • 修改現有程式碼
    • Bug 修復
    • 程式碼重構
  • 特色：
    • 直接操作檔案
    • 執行命令與測試
    • 即時驗證結果

🚀 Advanced 模式

用途：進階功能與整合

  • 適合場景：
    • 使用 MCP（Model Context Protocol）
    • 整合外部工具
    • 複雜的自動化任務
  • 特色：
    • 擴展 Bob 的能力
    • 支援客製化整合
    • 適合進階使用者

💡 模式切換技巧

在實際開發中，靈活切換模式能提升效率：

  1. Plan 模式：先規劃設計方案
  2. Code 模式：實作程式碼
  3. Plan 模式：遇到問題時重新分析
  4. Code 模式：繼續實作

🔧 2.2 Tool 使用機制

什麼是 Tool？

Bob 內建 12 個工具（Tools），讓 AI 能夠直接操作檔案、執行命令、分析程式碼等。這是 Bob 與傳統 AI 助手的最大差異。

工具類別 工具名稱 主要用途
檔案操作 read_file 讀取檔案內容
write_to_file 寫入完整檔案
apply_diff 精確修改檔案
insert_content 插入新內容
程式碼分析 list_files 列出檔案結構
search_files 搜尋檔案內容
list_code_definition_names 列出程式碼定義
執行與測試 execute_command 執行命令列指令
browser_action 瀏覽器操作
輔助功能 ask_followup_question 詢問使用者
attempt_completion 完成任務
update_todo_list 更新待辦清單

⚠️ 重要原則

Bob 每次只能使用一個工具，並且需要等待使用者確認結果後才能繼續。這確保了操作的可控性與安全性。

💬 2.3 對話流程與互動模式

典型的對話流程

  1. 使用者提出需求 
    請幫我建立一個 User Entity，包含 id、name、email 欄位
  2. Bob 分析並使用工具

    Bob 會先分析需求，然後使用適當的工具（如 write_to_file）

  3. 等待使用者確認

    工具執行後，Bob 會等待你確認結果

  4. 繼續或調整

    根據結果，繼續下一步或進行調整

💡 互動技巧

  • 明確的需求：清楚說明你要什麼
  • 提供上下文：說明專案背景與限制
  • 逐步確認：每個步驟都確認結果
  • 善用範例：提供參考範例更有效

🧠 2.4 Context 管理

什麼是 Context？

Context（上下文）是 Bob 理解你專案的基礎。包含：

  • 對話歷史
  • 已讀取的檔案內容
  • 專案結構資訊
  • Rules、Skills、AGENTS.md 等配置

⚠️ Context 的限制

Context 有大小限制（Token 限制），過多的 Context 會：

  • 增加成本（消耗更多 Token）
  • 降低回應速度
  • 可能影響準確性

💡 Context 管理策略

  1. 只讀取必要的檔案：使用 line_range 讀取部分內容
  2. 使用 Memory Import：匯入常用的知識（第 7 章詳述）
  3. 適時開新對話：完成一個任務後開新對話
  4. 善用 AGENTS.md：將專案知識寫入文件（第 6 章詳述）

✏️ 動手練習

練習 1：模式選擇

請判斷以下場景應該使用哪種模式：

  1. 需要設計一個新的資料庫結構 → Plan 模式
  2. 修復一個 Bug → Code 模式
  3. 評估兩種技術方案的優劣 → Plan 模式
  4. 重構一個複雜的函式 → Code 模式

練習 2：工具識別

請判斷以下任務需要使用哪個工具：

  1. 查看專案的檔案結構 → list_files
  2. 修改檔案中的某個函式 → apply_diff
  3. 建立一個新檔案 → write_to_file
  4. 搜尋所有包含 "TODO" 的檔案 → search_files
